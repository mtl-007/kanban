# PRD — 칸반보드 (Kanban Board)

## 1. 목적

팀 또는 개인이 작업을 시각적으로 관리할 수 있는 칸반보드를 제공한다. 별도 설치 없이 브라우저에서 즉시 사용 가능한 정적 웹 앱으로 시작하며, 향후 백엔드를 연동해 협업·영속성을 지원할 수 있도록 설계한다.

---

## 2. 사용자

| 페르소나 | 설명 |
|----------|------|
| 개인 개발자 | 혼자 작업 목록을 관리하고 싶은 사용자 |
| 소규모 팀 | 공유 보드로 진행 상황을 추적하는 팀 (향후) |

---

## 3. 핵심 기능 (MVP)

### 3.1 보드 구성
- 3개의 고정 컬럼: **TO-DO**, **In-Progress**, **Done**
- 각 컬럼 헤더에 카드 수 배지 표시

### 3.2 카드 관리
| 기능 | 설명 |
|------|------|
| 카드 추가 | 컬럼 하단 `+ 카드 추가` 버튼 → 제목 입력 후 추가 |
| 카드 삭제 | 카드 우측 상단 `×` 버튼 클릭 |
| 카드 이동 | 드래그 앤 드롭으로 다른 컬럼에 드롭 |

### 3.3 드래그 앤 드롭
- HTML5 Drag and Drop API 사용
- 드래그 중 카드 시각 피드백 (반투명 + 기울기)
- 드롭 대상 컬럼 하이라이트

### 3.4 반응형
- 768px 이하에서 컬럼 세로 배치

---

## 4. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 실행 환경 | 서버 없이 정적 파일만으로 동작 (빌드 도구 없음) |
| 배포 | GitHub Pages (`https://<username>.github.io/kanban/`) |
| 브라우저 지원 | Chrome / Firefox / Safari 최신 2버전 |
| 성능 | 초기 로딩 1초 이내 (정적 파일 기준) |
| 확장성 | 향후 RDB 백엔드(MySQL / PostgreSQL) 연동 가능한 구조 |

---

## 5. Phase 2: 사용자 인증 (구현됨)

Supabase Auth를 사용한 OAuth 2.0 기반 소셜 로그인.

| 기능 | 설명 |
|------|------|
| Google 로그인 | Supabase Google OAuth Provider |
| GitHub 로그인 | Supabase GitHub OAuth Provider |
| 세션 자동 복원 | Supabase SDK가 localStorage에 세션 저장, 새로고침 시 자동 복원 |
| 보드 접근 제어 | 미인증 시 전체 화면 로그인 오버레이 표시, 보드 숨김 |
| 사용자 정보 표시 | 헤더에 아바타 이미지 + 이메일 + 로그아웃 버튼 |

---

## 6. Phase 3: 데이터 영속성 (구현됨)

Supabase PostgreSQL DB에 카드 데이터를 저장해 새로고침 후에도 카드가 유지된다.

| 기능 | 설명 |
|------|------|
| 카드 로드 | 로그인 후 사용자 본인 카드를 DB에서 전체 조회 |
| 카드 추가 | DB insert 후 DOM 렌더링 (저장 중 버튼 비활성화) |
| 카드 삭제 | DB delete 후 DOM 제거 |
| 카드 이동 | 드래그 드롭 후 DB의 `column_name` 업데이트 (낙관적 업데이트) |
| 데이터 격리 | RLS 정책으로 사용자 본인 카드만 접근 가능 |
| 새로고침 | 세션 복원 → 카드 재로드로 데이터 영속 보장 |

---

## 7. 향후 기능 (Phase 4+)

- 카드 설명(description) 편집 모달
- 카드 순서 변경 (컬럼 내 드래그 정렬)
- 다중 보드 지원
- 팀 공유 및 실시간 협업

---

## 8. 제약사항

- 빌드 도구·서버 없음 (순수 정적 파일)
- 카드 제목 최대 50자
- Supabase anon key는 클라이언트에 노출 (공개 안전 키, RLS로 보안 관리)
- 컬럼은 3개(todo / inprogress / done)로 고정 (Phase 4+에서 동적 컬럼 검토)
